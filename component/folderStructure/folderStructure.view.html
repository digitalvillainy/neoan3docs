Folder Structure

Neoan3 foundation

Much like other frameworks Neoan3 has a specific folder structure that acts as the foundation to your project. While it may seem complex at first it's actually quite simple.

Folder Structure Overview

Below is a list of all of the files that you would typically see in a Neoan3 projects.

1. _neoan

2. _template

3. asset

4. component

5. frame

6. model

7. vendor

8. root level files

_neoan

This is the heart of Neoan3. Admittedly there isn't much of a reason to actually manipulate these files. Set up has already been taken care of for you and anything that needs to be customized specifically for this project exists somewhere else. Each file in _neoan could very well be it's own web page, so I inspire all of you to take some time and read through this folder. While it's not necessary in order to use the Framework, understanding how things work under-the-hood will allow for greater control over yoru project.

_template

You can influence the generated output with templates. To do so, place a folder _template in your project. Basic variable support works with double curly braces without spaces ( {{example}} ) The following files are respected:

Template File Entity Vairable Note
api.php component (API) name, frame
route.php component (Route) name, frame
view.html component (Route) name
ce.php Custom element name *PHP files for custom element is only generated if template is present
ce.js Custom element name
ce.html Custom element name
transformer.php Model transformer name, structure *Transformer usage requires neoan3-apps/transformer

asset

Simply put this contains all of the assets you would use on your site. Examples of this would be your favicon, images, files to be downloaded and svg's.

component

This is where most of the work is going to be done. This contains every component type that you are using for your project.

frame

Frames are a unique and vital piece to a Neoan3. Frames set up the primary styling, HTML, JavaScript. Using PHP it creates a primary location for the projects link tags, scripts, and any JavaScript that you need to run regardless of what component you're in.

model

While this folder won't be as used as much as the component folder it's purpose is vital. This contains the model.php for various components. Not every component needs a model, what's vital though to remember is that a model stands as a way to query the database for information. This is meant to be used across many different ctrl.php files. For example, Users.model.php could be used in user.ctrl.php but there is a case where a component named profile.ctrl.php may utilize it as well to gather the needed user information without needlessly creating a Profile.model.php.

vendor

Albeit that this is not a folder that is present in the default Neoan3 folders, it is a file that appears during composer installation and helps supports Neoan3 and it's apps.

Root Level Files

Many of the files contained on the root level will not be manipulated during the course of the project. Depending on what else you install in Neoan3 the files here may or may not change. But what remains as an important piece is the .htaccess file. During the Neoan3 installation process the Rewritebase portion of this file will be updated to the name of the project itself. However should there needed to be a an update this is the file that is often changed.
